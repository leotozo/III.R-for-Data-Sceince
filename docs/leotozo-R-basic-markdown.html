We can do vector operations in a very intuitive way.

vet1
[1] 1 2 3 4

When we do vet1 - 1, R subtracts 1 from each of the elements of the vector. The same happens when we do any arithmetic operation with vectors in R.

vet1 / 2
[1] 0.5 1.0 1.5 2.0
vet1 * 10
[1] 10 20 30 40

We can also do operations that involve more than one vector:

vet1 * vet1
[1]  1  4  9 16

In this case, R will align the two vectors and multiply element by element. This can get a little confusing when the two vectors are not the same size:

vet2 <- 1:3
vet1 * vet2
Warning in vet1 * vet2: longer object length is not a multiple of shorter
object length
[1] 1 4 9 4

The R aligned the two vectors and, as they did not have the same size, it was repeating the smaller vector until completing the larger vector. This behavior is called recycling and is useful for doing element-by-element operations (vectorized), but can sometimes be confusing.

DATE AND TIME

There are some functions for obtaining the current time and date.

Sys.time() - returns the absolute date / time that can be converted to multiple time zones and can return different days.
Sys.Date() - returns the current date in the current time zone.
Sys.timezone() - returns the time zone.
Sys.time()
[1] "2020-02-28 15:59:49 -03"
Sys.Date()
[1] "2020-02-28"
Sys.timezone()
[1] "America/Sao_Paulo"

Dates are represented by the Date class and the hours are represented by the POSIXct and POSIXlt classes.

Internally, dates are stored as the number of days since 1970-Jan-01 and times are stored as the number of seconds since 1970-Jan-01 (for POSIXct) or a list of seconds, minutes, hours, etc. (for POSIXlt).

Using the as.Date () function to convert strings to dates.

mydates <- as.Date(c("2007-06-22", "2004-02-13"))
mydates
[1] "2007-06-22" "2004-02-13"
class(mydates)
[1] "Date"

Performing operations between dates. Number of days between two dates.

days <- mydates[1] - mydates[2]
days
Time difference of 1225 days

The symbols below can be used with the format() function to return dates.

  • %d returns the day in number, example: 01-31 (month/day)

  • %a returns the abbreviated day of the week, example: Mon

  • %A returns the day of the week without abbreviation, example: Monday

  • %m returns the month, example: 00-12

  • %b returns the abbreviated month, example: Jan

  • %B returns the month without abbreviation, example: January

  • %y returns the year in 2-digits, example: 07

  • %Y returns the year in 4-digits, example: 2007

Printing todays date and using the format() function

today <- Sys.Date()

today
[1] "2020-02-28"
format(today, format="%B %d %Y")
[1] "February 28 2020"

Time Zones

An inherent feature of dates is the time zone. If we are working with event dates in Brazil and Scotland, for example, it is necessary to verify that the values follow the same time zone. In addition, when the exact date of an event is relevant, we may need to convert times to other time zones to communicate results in other countries.

To do these things, lubidate provides the with_tz() and force_tz() functions.

italian_GP <- ymd_hms("2019-09-08 15:10:00", tz = "Europe/Rome")
italian_GP
[1] "2019-09-08 15:10:00 CEST"
# Devolve qual seria a data em outro fuso

with_tz(italian_GP, tzone = "GMT")
[1] "2019-09-08 13:10:00 GMT"
with_tz(italian_GP, tzone = "America/Sao_Paulo")  
[1] "2019-09-08 10:10:00 -03"
# Altera o fuso sem mudar a hora

force_tz(italian_GP, tzone = "GMT")
[1] "2019-09-08 15:10:00 GMT"

 select() - select columns
